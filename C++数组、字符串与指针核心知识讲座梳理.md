# C++数组、字符串与指针核心知识讲座梳理
本次梳理基于三份核心笔记，按照**「基础数据结构→核心语法→进阶应用」**的讲座逻辑，将知识点系统化拆解，突出核心定义、关键规则、实操用法与高频易错点，形成完整的C++内存与数据操作知识体系，适配讲座的讲解节奏与听众理解路径。

## 一、第一模块：数组核心知识
数组是C++中最基础的线性数据结构，是后续指针、字符串操作的基础，核心围绕「连续内存、同类型元素」两大本质展开。

### 1. 数组基础定义
- 本质：**内存中一组连续的同类型存储区**，每个存储区称为数组的元素，同一数组所有元素类型完全相同，内存排布严格连续。
- 维数：由数组元素下标的个数决定，分为一维数组、二维数组，二维及以上统称为多维数组。
- 长度计算：`数组长度 = 数组占用总字节数 / 单个元素占用字节数`，通用公式为`sizeof(数组名)/sizeof(元素类型)`，一维、二维数组均适用。

### 2. 一维数组
一维数组是数组的基础形态，也是多维数组的组成单元。
#### （1）定义与初始化
- 定义格式：`元素类型 数组名[元素个数];`，数组下标从0开始，如`int a[3]`包含`a[0]、a[1]、a[2]`三个元素。
- 合法初始化方式：
  1.  全部初始化：`int ages[3] = {4,6,9};`
  2.  不指定长度的完全初始化：`int nums[] = {1,3,5,7};`，编译器自动根据元素个数确定数组长度。
  3.  部分初始化：`int nums[10] = {1,2};`，未显式初始化的元素，系统自动补0。
  4.  先定义后逐个赋值：`int nums[3]; nums[0]=1; nums[1]=2; nums[2]=3;`
- **高频易错点**：
  1.  禁止先定义再批量初始化：`int ages[3]; ages = {4, 6, 9};` 编译报错。
  2.  数组长度必须是编译时常量，禁止用变量定义长度：`int a=5; int arr[a] = {0};` 编译报错。
  3.  `const`修饰的数组，元素不可被修改，仅支持读取。

#### （2）访问与遍历
- 访问：通过下标（索引）访问，格式为`数组名[下标]`，支持读取与赋值。
- 遍历：通过for循环按下标有序访问每一个元素，核心场景包括数组元素求和、求最大值/最小值等。

### 3. 二维数组
#### （1）核心本质
二维数组是**元素为一维数组的一维数组**，概念上是二维的（行+列），但在物理内存中依然是连续的线性存储，按行依次存放，放完第一行再顺次放入第二行。

#### （2）定义与初始化
- 定义格式：`元素类型 数组名[行个数][列个数];`，如`int a[3][4]`定义了3行4列的整型数组，共12个元素。
- 合法初始化方式：
  1.  按行分段初始化：`int a[2][3]={{80,75,92},{61,65,71}};`
  2.  按行连续赋值：`int a[2][3]={80,75,92,61,65,71};`
  3.  省略第一维长度初始化：`int a[][3]={{1,2,3},{4,5,6}};`，第二维长度不可省略。
  4.  先定义后逐个赋值：通过`a[i][j]`格式逐元素赋值。

#### （3）访问与遍历
- 访问：通过双下标访问，格式为`数组名[行下标][列下标]`。
- 遍历：采用双层for循环，**外层循环遍历行（每一个一维数组），内层循环遍历列（一维数组的每个元素）**。
- 易错点：`a[0]、a[1]`是一维数组名，属于地址常量，不可作为下标变量随意修改。

## 二、第二模块：字符串核心知识
字符串是开发中最高频使用的数据类型，C++中分为**C风格字符串**和**C++ string类字符串**两类，核心区别在于内存管理与使用安全性。

### 1. 字符基础与前置知识
#### （1）char字符类型
- 本质：1字节的整型，用于存储单个字符，用英文单引号`' '`包裹，可参与算术运算。
- 核心特性：字符`'0'`的ASCII码值为48，大小写字母ASCII码值相差32，可通过加减32实现大小写转换。
- 输入方式：除`cin`外，可通过`getchar()`函数从键盘获取单个字符。

#### （2）核心转义字符
转义字符用于表示无法直接显示的特殊字符，核心高频转义字符如下：
| 转义字符  | 核心意义                    | ASCII码值 |
| --------- | --------------------------- | --------- |
| `\0`      | 字符串结束标记              | 000       |
| `\n`      | 换行，移至下一行开头        | 010       |
| `\r`      | 回车，移至本行开头          | 013       |
| `\t`      | 水平制表，跳到下一个TAB位置 | 009       |
| `\\`      | 反斜线字符本身              | 092       |
| `\'` `\"` | 单引号、双引号字符本身      | 039、034  |

### 2. C风格字符串（字符数组实现）
#### （1）核心定义
**以`'\0'`（数字0）结尾的char数组，才是合法的C风格字符串**；若char数组没有以`'\0'`结尾，仅为普通字符数组，不是字符串。

#### （2）定义与初始化
- 合法初始化方式：
  1.  双引号直接赋值：`char c[] = "abc";`，编译器自动在末尾补`'\0'`，数组长度为4。
  2.  大括号带结束符初始化：`char c[3] = { 'a','b','\0'};`。
  3.  部分初始化：`char c[5] = {'a', 'b', 'c'};`，未初始化的元素自动补0（即`'\0'`），属于合法字符串。
- **高频易错点**：`char c[3] = "abc";` 编译报错，因为字符串需要额外1字节存储`'\0'`，数组容量不足。

#### （3）核心操作
1.  **输入输出**：可直接通过`cout/printf`输出数组名，`cin/scanf`输入到数组中（需注意数组越界）；更安全的输入方式为`fgets(数组名, 数组长度, stdin)`，可限制输入长度，避免缓冲区溢出。
2.  **长度计算**：`strlen(字符串)`函数，返回有效字符个数，**不包含末尾的`'\0'`**；注意与`sizeof()`区分：`sizeof()`返回数组总占用字节数，包含`'\0'`。
3.  **访问与遍历**：通过下标访问单个字符；遍历可通过for循环下标遍历，或通过指针以`'\0'`为结束标记循环遍历。
4.  **常用库函数**：
    | 函数                       | 功能                           | 注意事项                                        |
    | -------------------------- | ------------------------------ | ----------------------------------------------- |
    | `strcat(dest, src)`        | 将src字符串拼接到dest末尾      | 需保证dest有足够内存，否则会缓冲区溢出          |
    | `strcmp(s1, s2)`           | 按ASCII码逐字符比较两个字符串  | 相等返回0，s1>s2返回正数，s1<s2返回负数         |
    | `strcpy(dest, src)`        | 将src字符串复制到dest中        | 需保证dest有足够内存，推荐使用安全版本`strncpy` |
    | `strstr(haystack, needle)` | 在主串中查找子串首次出现的位置 | 找到返回对应位置指针，未找到返回NULL            |

### 3. C++ string类字符串（推荐使用）
C++标准库提供的字符串类，底层基于字符数组实现，封装了内存管理，无需手动处理`'\0'`，无数组越界风险，是开发首选方案。
- 定义与初始化：`std::string s = "abc";`
- 核心操作：
  1.  输入输出：直接通过`cin/cout`操作字符串名。
  2.  长度计算：通过`.length()`或`.size()`方法获取有效字符长度，不包含`'\0'`。
  3.  访问与遍历：通过下标`[i]`访问单个字符，for循环下标遍历。
  4.  常用操作：直接用`+`/`+=`实现字符串拼接，用`>`/`<`/`==`等运算符直接比较字符串，用`=`直接赋值复制，用`.find()`方法查找子串（未找到返回`string::npos`）。

## 三、第三模块：指针基础原理
指针是C++的核心灵魂，本质是对内存地址的直接操作，是实现数组、字符串、函数高级用法的底层基础。

### 1. 指针核心概念
- 地址：内存中每个存储单元的唯一编号，通过`&变量名`可获取变量的内存地址；数组的元素在内存中连续排布，地址依次递增。
- 指针：即内存地址，“指向”变量/数组元素的存储单元；**指针变量**是专门用于存储其他变量地址的变量，简称指针。

### 2. 指针变量核心规则
#### （1）定义与初始化
- 定义格式：`数据类型 * 指针变量名;`，数据类型必须与指向的变量类型一致。
- 合法初始化方式：
  1.  定义同时初始化：`int a = 1; int *pa = &a;`，`&`为取地址符，获取变量的内存地址。
  2.  先定义后赋值：`int a = 1; int *pa; pa = &a;`
- 核心特性：多个指针可指向同一个地址，指针的指向可随时修改。

#### （2）指针的大小
指针的大小仅与系统平台相关，与指向的数据类型无关：
- 32位系统：所有指针均为4字节；
- 64位系统：所有指针均为8字节。

#### （3）核心操作：解引用
- 符号`*`有两种核心含义：
  1.  定义变量时：`*`是类型说明符，表示该变量是指针变量；
  2.  定义完成后：`*`是解引用符，用于访问指针指向的内存空间的内容，可通过解引用直接修改对应变量的值。
- 示例：`int a=1; int *pa=&a; *pa=10;` 执行后，变量a的值被修改为10。

#### （4）野指针与空指针（重点避坑）
1.  **野指针**：未初始化的指针，内部是随机的垃圾值，**绝对禁止解引用**，会导致程序崩溃，开发中严禁出现野指针。
2.  **空指针**：初始化为`NULL`（或0）的指针，`int *p = NULL;`；空指针不可直接解引用，重新绑定有效内存地址后，方可正常使用。

#### （5）const修饰的指针（易混淆考点）
根据const的位置不同，分为两种核心类型，规则完全不同：
| 类型     | 定义格式                          | 核心规则                                                     |
| -------- | --------------------------------- | ------------------------------------------------------------ |
| 常量指针 | `const int *pa` / `int const *pa` | 不可通过指针修改指向变量的值，但可以修改指针本身的指向（可指向其他变量）；变量本身可直接修改 |
| 指针常量 | `int * const pa`                  | 可以通过指针修改指向变量的值，但不可修改指针本身的指向（不可再指向其他变量） |

#### （6）void* 万能指针
- 特性：可指向任意类型变量的内存空间，无固定类型。
- 使用规则：不可直接解引用，必须先强制转换为对应类型的指针，才能操作指向的内存。

## 四、第四模块：指针进阶应用
指针的核心价值在于与数组、函数、字符串的结合，也是C++开发的核心难点与高频考点，本模块将三者的联动规则彻底梳理清晰。

### 1. 指针与数组
#### （1）核心底层规则
**数组名是数组首元素的地址**，与`&数组名[0]`完全等价，数组名本质是一个地址常量。

#### （2）一维数组与指针
- 数组指针定义：`int a[5] = {1,2,3,4,5}; int* pa = a;`，指针pa指向数组首元素。
- 元素访问：两种等价方式，**下标法`pa[i]`** 与 **指针法`*(pa+i)`**，完全等效。
- 指针加减运算：指针`+1`，实际偏移的是「1个对应数据类型的字节数」，如`int*`+1偏移4字节，`double*`+1偏移8字节；支持`++`/`--`/`+=`/`-=`运算。
- 数组遍历：可通过指针偏移实现遍历，**注意：数组名是地址常量，不可进行`++`/`--`等修改操作**。

#### （3）二维数组与指针
- 二维数组的地址本质：`int arr[m][n]` 是由m个长度为n的一维数组组成，`arr`是首行一维数组的地址，而非单个元素的地址。
- 行指针（数组指针）定义：`int (*p)[n] = arr;`，指向一个包含n个int元素的一维数组，与二维数组的行维度匹配。
- 元素访问：`p[i][j]` 等价于 `*(p[i]+j)` 等价于 `*(*(p+i)+j)`。
- 指针加减运算：行指针`+1`，直接偏移一整行的字节数（n×元素类型字节数），跳到下一行首地址。
- 遍历方式：① 双层for循环，通过行指针遍历行、再遍历列；② 用`int* ptr=&arr[0][0]`，通过单层循环遍历二维数组的连续物理内存。

#### （4）指针数组
- 本质：**数组**，数组的每个元素都是指针变量。
- 定义格式：`int* p_arr[5];`（整型指针数组）、`const char* p_str[5];`（字符指针数组）。
- 核心用法：字符指针数组可存储多个字符串的首地址，相当于特殊的二维数组，可通过下标直接访问对应字符串。

### 2. 指针与函数
指针与函数结合，核心解决了「函数内修改实参值」「数组批量传参」「函数动态调用」三大核心问题。

#### （1）函数参数的三种传递方式
| 传递方式 | 核心原理                                                     | 对实参的影响                             |
| -------- | ------------------------------------------------------------ | ---------------------------------------- |
| 值传递   | 实参将值复制给形参，形参与实参占用不同内存                   | 形参的任何修改，都不会影响实参的值       |
| 地址传递 | 用指针/数组名作为参数，传递实参的内存地址，形参与实参共享同一块内存 | 形参中对指向内容的修改，会直接同步到实参 |
| 引用传递 | C++特有特性，给变量起别名，共享同一块内存                    | 形参修改会直接影响实参                   |

#### （2）数组名作为函数参数
- 核心规则：**数组名作为函数参数时，会自动退化为指针**，函数内通过`sizeof(形参数组名)`获取的是指针的大小（64位系统8字节），而非数组总大小。
- 正确用法：将数组名传入函数的同时，必须同步传入数组长度，才能在函数内正确遍历数组。

#### （3）指针函数
- 本质：**函数**，返回值是一个指针（地址）。
- 致命易错点：**绝对不能返回函数内局部变量的地址**，局部变量在函数执行结束后生命周期终止，内存被释放，会导致野指针，引发程序崩溃；仅可返回全局变量、静态变量的地址。

#### （4）函数指针
- 本质：**指针**，指向函数的指针，函数名本身就是函数的入口地址。
- 定义格式：`返回值类型 (*指针变量名)(形参类型列表);`，示例：`int (*p)(int,int) = sum;`（sum是一个接收两个int参数、返回int的函数）。
- 核心用法：通过函数指针调用对应函数，可实现函数的动态调用，典型场景为通过函数指针实现四则运算、回调函数等。

### 3. 指针与字符串
字符串的底层是字符数组，指针是操作字符串的核心方式，C语言字符串库函数的底层均基于指针实现。

#### （1）字符指针定义字符串
- 格式：`const char* p_str = "abc";`，字符串字面量存储在只读常量区，**必须加const修饰**，禁止通过指针修改字符串内容，否则会触发内存异常。

#### （2）字符数组 vs 字符指针定义字符串
| 核心特性 | `char str[] = "abc";`  | `const char* p_str = "abc";`         |
| -------- | ---------------------- | ------------------------------------ |
| 存储位置 | 栈区，可读可写         | 只读常量区，不可修改内容             |
| 操作权限 | 可修改数组内的单个字符 | 不可修改字符串内容，可修改指针的指向 |
| 内存占用 | 数组总长度（含'\0'）   | 指针占8字节（64位）+ 常量区字符串    |

#### （3）指针遍历字符串
核心原理：利用C风格字符串以`'\0'`结尾的特性，无需提前知道字符串长度，通过指针从首地址开始，循环判断`*p != '\0'`，每次`p++`即可遍历所有字符，是字符串操作的最底层实现方式。

## 五、讲座核心总结与知识闭环
本次知识体系的核心逻辑，是围绕**「C++内存操作」**形成完整闭环：
1.  数组是「连续内存的同类型数据集合」，是数据存储的基础形态；
2.  字符串是「特殊的字符数组」，是业务开发中最常用的数据形态；
3.  指针是「内存地址的操作工具」，是C++直接操控内存的核心能力；
4.  指针与数组、函数、字符串的结合，实现了数据的高效传递、灵活操作与底层控制，是C++区别于其他高级语言的核心优势。

讲座讲解过程中，可围绕「定义→用法→示例→易错点」的节奏展开，重点突出实操案例与避坑指南，帮助听众彻底理解C++内存操作的底层逻辑，而非单纯记忆语法规则。
